Overview

Load Limit is a browser-based structural engineering simulator that models 2D truss bridge behavior under dynamic vehicular loading. The simulation uses Position-Based Dynamics (PBD) with Verlet integration to compute joint displacements, and an iterative constraint relaxation solver to maintain structural connectivity.

The goal is to provide an interactive environment where users explore fundamental structural engineering concepts — load paths, material selection, truss geometry, resonance, and failure modes — without requiring engineering prerequisites.

Simulation Method

Position-Based Dynamics (Verlet integration) with 8 constraint relaxation passes per frame at 60 fps.

Failure Detection

Sag-based: joint vertical displacement from settled baseline exceeds material-specific threshold.

Load Model

Vehicle weight distributed to nearby structural nodes via linear proximity weighting.

Physics Scale

1 pixel = 0.05 meters. Bridge spans range from 16m (Narrow Gorge) to 28m (Grand Canyon).

Physics Model

Verlet Integration

Each free joint in the structure is simulated as a point mass. Position is updated using Verlet integration, which implicitly encodes velocity as the difference between current and previous position:

x(t+Δt) = x(t) + (x(t) − x(t−Δt)) · d + F · Δt²

where:
  x(t) = current position
  x(t−Δt) = previous position (velocity is implicit)
  d = damping coefficient (0.96 per frame)
  F = net external force (gravity + vehicle load + wind)
  Δt = 1/60 s (one frame at 60fps)

This approach is numerically stable for constraint-heavy systems and naturally conserves energy without explicit momentum tracking.

Constraint Relaxation

Structural beams are modeled as distance constraints between pairs of joints. Each frame, the solver runs 8 iterations of constraint projection — moving joint pairs toward their rest-length distance:

correction = (|p_B − p_A| − L₀) / |p_B − p_A| · 0.5

p_A += correction · (p_B − p_A) [if A is free]
p_B -= correction · (p_B − p_A) [if B is free]

where L₀ = rest length of the beam segment

Fixed anchor joints are immovable (infinite mass approximation). Free joints receive equal correction from both ends unless one is fixed, in which case the free joint absorbs the full correction.

Beam Subdivision

Each user-drawn beam is subdivided into segments of approximately 38px (≈1.9m) with free intermediate joints. This allows beams to exhibit realistic bending and sag under distributed load, rather than behaving as perfectly rigid rods.

Resonance Model

A resonance accumulator increases as vehicles remain on the bridge span. Above a threshold, a sinusoidal horizontal forcing function is applied to all free joints, simulating the self-excited oscillation mechanism that caused the 1940 Tacoma Narrows Bridge collapse:

F_resonance(t) = R · A · sin(ω · t)

where R = resonance level (0–1), A = 0.25px, ω = 0.12 rad/frame
Failure stress is amplified by factor (1 + R · 0.6)

Cable material: when a constraint is in compression (current length < rest length), the correction step is skipped. This correctly models tension-only members that go slack under compressive load — the physical behavior of real suspension cables.

Failure Criteria

A beam segment fails when the maximum vertical sag of its endpoint joints (measured from the post-settlement baseline) exceeds the material's sag limit:

sag_A = max(0, y_A(t) − y_A(settled))
sag_B = max(0, y_B(t) − y_B(settled))

failure if: max(sag_A, sag_B) · resonanceFactor > sagLimit_material

where resonanceFactor = 1 + R · 0.6

Sag from the settlement baseline (rather than from zero) is used so that the natural deflection of the structure under its own weight does not trigger false failures — only load-induced deflection beyond the resting state is penalized.

Rain reduces effective sag limits by 15% (modeling reduced material strength under wet conditions). Foundation cracking is tracked separately as accumulated anchor load exceeding a threshold.

Tension vs Compression

Beam mode is tracked each frame: if the current inter-joint distance exceeds rest length, the member is in tension (being pulled apart). If shorter, it is in compression (being squeezed). The Force Diagram overlay renders these states with directional arrows. Identifying which members carry which load type is fundamental to efficient truss design — tension members can be cables, compression members require stiffer materials.

Load Model

Vehicle weight is applied as downward impulse to nearby structural joints each frame. The force is distributed proportionally by horizontal proximity:

proximity_i = 1 − |x_joint − x_vehicle| / (vehicle_width · 0.75)

F_i = F_vehicle · (proximity_i / Σ proximity_j)

where the sum is over all free joints within the vehicle's footprint

This approximates a distributed load model — heavier, wider vehicles (train, tank) spread load over more joints than narrow vehicles (bicycle, sedan). Vehicle Y-position follows the bridge surface, detected by finding the nearest non-cable beam segment above the vehicle's horizontal position.

Known Limitations

⬛ 2D planar analysis only. Out-of-plane effects, lateral buckling, and torsion are not modeled. Real bridges require 3D analysis.
⬛ Simplified material model. No fatigue, plastic deformation, or creep. Real materials degrade over time and fail differently under cyclic loading.
⬛ Approximated joint masses. Node mass is computed from attached beam geometry rather than from explicit member cross-sections.
⬛ Simplified dynamics. The constraint solver is not a true finite element model. No modal analysis, no natural frequency computation, no damping coefficients.
⬛ Linear proximity load distribution. Real bridge decks distribute loads through complex plate behavior, not simple horizontal proximity weighting.
⬛ No soil-structure interaction. Foundation conditions and soil deformation are not modeled — anchors are treated as perfectly rigid.
